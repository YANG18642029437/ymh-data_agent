# Expo 高级开发规范

本 Skill 包含 Expo/React Native 项目中的高级开发模式与规范。

## 使用方式

分析用户需求后，根据下方「场景匹配规则」判断需要读取哪些参考文档。
一个任务可能匹配多个场景，按需读取多个文档。

## 场景匹配规则

### 1. 瀑布流布局 → `references/masonry-layout.md`

**判断条件**（满足任一）：
- 用户明确要求瀑布流、Pinterest 风格、双列不等高
- 需求涉及的产品形态天然使用瀑布流（如：小红书、花瓣、Unsplash、商品橱窗、图片社区）
- 要求"卡片高度不一致"、"交错排列"、"图片流"

### 2. 文件上传与处理 → `references/file-upload.md`

**判断条件**（满足任一）：
- 用户可以上传图片、视频、音频、文件到服务器
- 涉及头像设置、图片发布、附件上传、拍照提交
- 前端获取的本地文件需要传递给后端处理
- 需要在前端页面间传递图片/文件数据

### 3. 音频录制与播放 → `references/audio-record-play.md`

**判断条件**（满足任一）：
- 需要录音功能（语音输入、语音消息、录音笔等）
- 需要播放音频（音乐播放、语音播放、音效等）

### 4. 保存图片到相册 → `references/save-to-gallery.md`

**判断条件**（满足任一）：
- 用户需要将图片下载/保存到手机相册
- 涉及"保存到本地"、"下载图片"功能
- AI 生成的图片需要提供保存功能

### 5. 顶部 Tab 导航 → `references/top-tab-navigation.md`

**判断条件**（满足任一）：
- 某个页面内部需要顶部可切换的子分类（如：关注/推荐/热榜）
- 需求涉及的产品形态使用顶部 Tab（如：抖音首页、微博、B站、新闻类首页）
- 注意：这是**页面内组件级** Tab，不是底部导航栏（底部 Tab 用 Expo Router 的 `(tabs)` 目录实现，不需要本文档）
